# دليل استخدام OMRAN ERP للمدير

هذا الدليل يشرح طريقة استخدام النظام من وجهة نظر المدير، بدون تفاصيل تقنية. الهدف هو توضيح كيف يبدأ المدير مشروعاً جديداً، يحول المحادثة إلى عرض، يحدد الورش، يتابع التنفيذ، ويراجع حالة المشروع بالذكاء الاصطناعي.

## 1. فكرة النظام باختصار

OMRAN هو نظام mini ERP موجه لإدارة مشاريع الترميم والبناء، مع دعم من الذكاء الاصطناعي.

يساعد المدير على:

- تسجيل مشروع جديد من خلال محادثة ذكية.
- تحويل كلام العميل أو المدير إلى عرض منظم.
- تحديد مواقع العمل داخل المشروع.
- اختيار الورش المناسبة لكل موقع.
- متابعة التنفيذ بالصور والمهام والمشاكل والمواد.
- مراجعة حالة المشروع من خلال AI Monitoring.
- إنشاء الفواتير ومتابعة الدفعات.

## 2. بدء مشروع جديد من AI Intake

افتح صفحة:

```text
AI Intake
```

ثم اتبع الخطوات التالية:

1. اضغط زر `New` أو `جديد` لإنشاء سجل استقبال جديد.
2. اكتب معلومات المشروع بشكل طبيعي، مثل:
   - اسم الشركة أو العميل.
   - العنوان والموقع.
   - الشخص المسؤول ورقم الهاتف والبريد.
   - تاريخ البداية والنهاية المتوقع.
   - مناطق العمل المطلوبة.
3. يمكن استخدام الكتابة أو الصوت إذا كانت ميزة Voice-to-Text مفعلة.
4. المساعد الذكي سيطرح أسئلة فقط عن المعلومات الناقصة.

مثال رسالة بداية:

```text
عندي مشروع ترميم لشركة إعمار الشام. الموقع في دمشق، حي المزة، بناء رقم ١٥. المسؤول أحمد منصور، الهاتف ٠٩٣٣٤٤٥٥٦٦، والبريد ahmad@example.com. بدنا نبدأ ١٠-٠٦-٢٠٢٦ وننتهي ٢٥-٠٦-٢٠٢٦.
```

## 3. إدخال نطاق العمل

بعد تسجيل معلومات المشروع الأساسية، أدخل تفاصيل العمل حسب المناطق.

مثال:

```text
المطبخ يحتاج إزالة البلاط القديم وتركيب سيراميك ٦٠ في ٦٠ مع إصلاح تمديدات المياه وترميم خزائن خشبية. الحمام الرئيسي يحتاج عزل مائي سائل مرن وتبديل مغسلة وخلاط دش. غرفة الجلوس تحتاج دهان داخلي عادي مع معجون كامل وطبقتين.
```

النظام يحاول استخراج:

- اسم الموقع أو منطقة العمل.
- نوع العمل المطلوب.
- المهارات أو الورش المطلوبة.
- المواد أو التفاصيل المهمة.
- المعلومات الناقصة التي تحتاج تأكيد.

## 4. توليد العرض Proposal

بعد أن تصبح معلومات المشروع واضحة:

1. اضغط زر `Generate Proposal` أو `إنشاء العرض`.
2. النظام يحول المحادثة إلى بيانات منظمة.
3. راجع الحقول التالية:
   - معلومات العميل.
   - عنوان المشروع.
   - ملخص المشروع.
   - مواقع العمل.
   - المهارات المطلوبة.
   - الورش المقترحة أو المطلوبة.
   - الدفعات والمبلغ المتوقع.
4. إذا وجدت خطأ، عدل الحقول يدوياً قبل التأكيد.

مهم: الذكاء الاصطناعي يساعد في التنظيم، لكن المدير يجب أن يراجع العرض قبل اعتماده.

## 5. تحديد الورش Workshops

النظام يعتمد على الورش الخارجية كموديل تنفيذ رئيسي.

من صفحة AI Intake أو Order Detail:

1. راجع كل موقع عمل.
2. حدد الورشة المسؤولة عن الموقع.
3. تأكد من المهارات التي تغطيها الورشة، مثل:
   - بلاط.
   - عزل مائي.
   - دهان.
   - كهرباء.
   - صحية.
4. إذا لم توجد ورشة مناسبة، اترك الموقع كـ `Workshop needed` أو أضف ورشة جديدة من صفحة Workshops.

## 6. إدارة الورش

افتح صفحة:

```text
Workshops
```

يمكن للمدير:

- إضافة ورشة جديدة.
- تعديل بيانات الورشة.
- تحديد الاختصاصات.
- تحديد حالة التوفر: Available أو Not Available.
- إضافة ملاحظات عن الورشة.

مهم: النظام لا يقترح الورش غير المتاحة ضمن الاقتراحات التنفيذية.

## 7. تحويل العرض إلى طلب Order

بعد مراجعة العرض والورش:

1. اضغط `Confirm` أو `Convert to Order`.
2. النظام ينشئ طلباً رسمياً.
3. يمكن فتح الطلب من صفحة Orders.

بعد التحويل، يصبح المشروع قابلاً للتتبع من صفحات:

- Order Detail.
- Project Tracking.
- AI Monitoring.
- Invoices.

## 8. متابعة المشروع Project Tracking

افتح الطلب من صفحة Orders، ثم اضغط:

```text
Open Tracking
```

صفحة Project Tracking تستخدم للتوثيق اليدوي للتنفيذ.

يمكن للمدير متابعة:

- Progress Overview.
- Baseline schedule.
- Timeline updates.
- Photos.
- Tasks.
- Issues.
- Materials.
- Workshops schedule.

## 9. إضافة تحديثات التنفيذ

في Tracking، يمكن إضافة تحديث جديد يحتوي على:

- الموقع أو منطقة العمل.
- عنوان التحديث.
- الوصف.
- الحالة.
- نسبة الإنجاز.
- الإجراء التالي.
- صور من الموقع.

مثال:

```text
تم الانتهاء من إزالة البلاط القديم في الحمام، وتجهيز الأرضية للعزل المائي. الإجراء التالي: تنفيذ طبقة العزل الأولى.
```

## 10. إدارة المهام Tasks

لكل موقع يمكن إضافة مهام، مثل:

- إزالة البلاط القديم.
- تنفيذ العزل المائي.
- تركيب السيراميك.
- تنظيف نهائي.

كل مهمة يمكن أن تحتوي على:

- اسم المهمة.
- الحالة.
- المسؤول.
- تاريخ الاستحقاق.
- ملاحظات.
- وزن المهمة.
- نسبة تقدم المهمة.

الوزن يعني أهمية المهمة ضمن تقدم الموقع. مثلاً إذا كانت مهمة العزل مهمة جداً يمكن أن تأخذ وزن أعلى من مهمة التنظيف.

## 11. إدارة المشاكل Issues

إذا ظهر عائق أثناء التنفيذ، أضفه من تبويب Issues.

مثال:

```text
تأخر تسليم السيراميك من المورد، وهذا قد يؤثر على موعد تنفيذ الحمام.
```

كل مشكلة تحتوي على:

- العنوان.
- الوصف.
- الشدة: Low / Medium / High.
- الموقع.
- المسؤول.
- الحالة: Open / In Progress / Resolved.
- ملاحظة الحل.

المشاكل المفتوحة تظهر ضمن التحذيرات وتؤثر على AI Monitoring.

## 12. متابعة المواد Materials

من تبويب Materials يمكن تسجيل المواد المهمة، مثل:

- سيراميك.
- دهان.
- مواد عزل.
- أدوات صحية.
- تمديدات كهرباء أو صحية.

كل مادة تحتوي على:

- اسم المادة.
- الكمية.
- الموقع.
- الحالة: Needed / Ordered / Delivered / Used.
- ملاحظات.

## 13. الجدول الأساسي Baseline

Baseline هو الخطة الزمنية الرسمية للمواقع داخل المشروع.

طريقة الاستخدام:

1. افتح Tracking.
2. ادخل إلى تبويب Baseline.
3. يمكن طلب اقتراح مبدئي من النظام.
4. راجع تواريخ البداية والنهاية لكل موقع.
5. عدل التواريخ إذا لزم الأمر.
6. اضغط Confirm لاعتمادها.

مهم: التحليل الرسمي للتأخير يعتمد على Baseline مؤكد، وليس مسودة فقط.

## 14. AI Monitoring

افتح الطلب ثم اضغط:

```text
Open AI Monitoring
```

AI Monitoring يراجع بيانات التتبع ويشرح حالة المشروع.

يعرض:

- التقدم المخطط Planned Progress.
- التقدم الفعلي Actual Progress.
- الفرق بين المخطط والفعلي.
- حالة التأخير.
- تاريخ الانتهاء المتوقع.
- المشاكل المفتوحة.
- المعلومات الناقصة.
- الإجراءات المقترحة للمدير.

مهم: الذكاء الاصطناعي لا يخترع أرقاماً رسمية. هو يشرح القيم التي حسبها النظام من بيانات التتبع.

## 15. الفواتير Invoices

افتح صفحة:

```text
Invoices
```

يمكن للمدير:

- إنشاء فاتورة.
- مراجعة بنود الفاتورة.
- تعديل الحالة.
- تصدير PDF أو Word.
- متابعة الدفعات.

الفواتير في هذا الاتجاه تعتمد على المشروع والمواقع والبنود، وليس على ساعات موظفين داخليين.

## 16. أفضل طريقة استخدام يومية

الاستخدام اليومي المقترح للمدير:

1. افتح Dashboard لمراجعة الوضع العام.
2. افتح Orders للمشاريع النشطة.
3. افتح Tracking للمشروع المطلوب.
4. أضف تحديثات وصور ومهام ومشاكل.
5. راجع AI Monitoring لمعرفة التأخير والمخاطر.
6. حدث الورش أو المواد حسب الحاجة.
7. راجع الفواتير والدفعات.

## 17. أخطاء شائعة وطريقة التعامل معها

| الحالة | السبب | الحل |
| --- | --- | --- |
| AI يطلب معلومات ناقصة | بعض تفاصيل المشروع غير واضحة | أجب بتفاصيل قصيرة ومباشرة. |
| لا تظهر ورشة مناسبة | لا توجد ورشة متاحة بنفس الاختصاص | أضف ورشة جديدة أو عدل حالة توفر الورشة. |
| AI Monitoring لا يعطي تحليل قوي | لا توجد مهام أو Baseline أو تحديثات كافية | أضف مهام، نسب تقدم، Baseline، ومشاكل إن وجدت. |
| التحذير لا يختفي | السبب الأصلي لم يتم إصلاحه بعد | افتح زر المعلومات بجانب التحذير ونفذ الإجراء المطلوب. |
| PDF لا يظهر في المتصفح | المتصفح أو الأداة لا تعرض PDF مباشرة | استخدم download أو افتحه في تبويب جديد. |

## 18. قاعدة مهمة

النظام يساعد المدير على التنظيم واتخاذ القرار، لكنه لا يستبدل مراجعة المدير.

قبل اعتماد أي عرض أو فاتورة أو خطة تنفيذ، يجب مراجعة:

- الأسعار.
- التواريخ.
- نطاق العمل.
- الورش المختارة.
- الدفعات.
- التحذيرات.
